Discuss the scope of var, let, and const

var: Function-scoped, hoisted, and can be redeclared and reassigned. Avoid using var in modern JavaScript as it can lead to unexpected behavior.
let: Block-scoped, not hoisted, and can be reassigned. It's suitable for variables that need to change their values within a block.
const: Block-scoped, not hoisted, and cannot be reassigned. Use const for variables that should never change their value after assignment. However, note that the content of objects and arrays assigned to const can be modified.

Tell us the use cases of null and undefined

null: Typically used by developers to explicitly represent the absence of a value or to reset a variable's value to nothing.
undefined: Often represents the unintentional absence of a value, typically when a variable is declared but not assigned a value. It's also the default return value of functions without a return statement.

What do you mean by REST API?

REST API (Representational State Transfer Application Programming Interface) is a standardized way for software applications to communicate and exchange data over the internet using standard HTTP methods. It follows a set of architectural principles, such as statelessness, resource-based URLs, and HTTP verbs, to enable easy and scalable interaction between different systems and services.
